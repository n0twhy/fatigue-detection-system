## 一、师兄论文可借鉴之处（邹秉航《面向工业环境的无干扰式多模态人员疲劳检测系统》，川大硕士 2026）

**它做的**：面部视频 + 柔性压力坐垫 + 智能手环三模态；面部提头姿/眼嘴形态/**面部动作单元
AU**/**运动频率**/rPPG 心率HRV；极端随机树选特征；多分支网络+logit 决策融合（二分类 93%）；
用 **Karolinska 嗜睡量表(KSS)**+Oddball 范式标注；创新点是 **KAN(Kolmogorov-Arnold 网络)**
做三级分类。

**我们能借鉴的（符合单摄像头+不训练）**：
| 师兄的点 | 我们怎么用 | 采纳 |
|---|---|---|
| **KSS 嗜睡量表标注** | 把融合分映射到国际公认的 KSS 1-9，给四级判定科学背书 | ✅ 实现（见创新④） |
| 运动频率特征 | 头/眼微动频率——与眨眼动力学合并 | ✅ 并入创新② |
| 面部动作单元 AU | 疲劳相关 AU（闭眼AU43/眨眼AU45/哈欠）——**需训练/额外模型**，超单摄轻量范围 | 📌 未来工作 |
| 多分支网络 + logit 决策融合 | 我们做**加权融合**（同为决策级），并升级为**质量感知加权**（创新①）| ✅ 升级 |
| KAN 三级分类网络 | **需训练**，违反铁律 | 📌 未来工作 |
| 压力坐垫 / 手环多传感器 | 无硬件、非单摄 | 📌 未来工作/展望 |

**结论**：师兄的"多模态决策融合 + 分级 + KSS 标注"思路我们**部分继承并轻量化**；他的重型
部分（多传感器硬件、AU 模型、KAN 训练）作为报告"未来工作/对比"，正好体现三点延伸中的
"极简单摄像头实时化"与"传统 vs 深度对比"。

## 三、任务书约束核对（严格遵守）

- 基础任务：阈值判断 + 加权融合，划分清醒/轻度/中度/重度四级——**我们的创新都建立在此之上，
  不改基础模型形态**（仍是可解释的加权融合，只是权重变自适应）。
- 防误报策略（连续帧/滑窗/评分平滑/多条件）——创新③正是其自适应升级。
- 不强制公开数据集——创新点用 CEW/YawDD 做验证属加分。
- 六区界面——KSS 显示并入③疲劳等级区，不破坏布局。

## 四、本次实现清单（保留现有进度，新增为创新点）

1. **创新① 质量感知自适应融合**（`core/fusion.py`）：各子分权重 × 实时可靠度（头转→眼部降权、
   HR 无/SNR 低→生理降权、人脸置信度低→整体降权），再归一化。**最核心的创新**。
2. **创新② 眨眼动力学**（`core/feature_window.py` + `types.py`）：平均眨眼时长、微睡眠（连续
   闭眼 >0.5s）计数，纳入眼部子分与显示。
3. **创新③ 置信度自适应决策窗口**（`core/fusion.py` AlarmFSM）：子分一致且高时缩短报警所需
   连续窗口、矛盾时延长，兼顾灵敏与防误报。
4. **创新④ KSS 嗜睡量表对齐**（`core/fusion.py` + UI）：融合分 → KSS 1-9，等级区并列显示。

**未来工作（报告展望，体现对领域全貌的把握）**：面部动作单元 AU、KAN/深度分级、多传感器
（坐垫/手环）多模态——均因"单摄像头轻量实时 + 不训练"的定位而不纳入本系统，正是与师兄
重型离线系统的差异化。

### 实现记录（2026-07-13，v1.8）

四个创新点代码全部落地，单元测试 11 条断言全绿（`dev_tools/verify_innovations.py`），
全回归无回退（M3 融合 25 项、M4 rPPG 12 项、低帧率、弱 EAR、组员反馈 10 项、CEW 96.6%）。

**创新① 质量感知自适应融合**
- `core/fusion.py :: reliabilities(wf, cfg)`：由信号质量算各子分实时可靠度——
  `eye = face_ratio × (1 − min(yaw_penalty_cap, mean_abs_yaw/yaw_reliability_deg))`，
  `mouth/head = face_ratio`，`physio = 1.0`。
- `core/fusion.py :: fuse(sub, weights, reliabilities)`：有效权重 = 固定权重 × 可靠度，
  再归一化（`S = Σ(w·r·s)/Σ(w·r)`）；`reliabilities=None` 时退化为原固定权重融合。
- 特征来源 `core/feature_window.py :: _compute_signal_quality()`：近窗人脸检出帧占比
  `face_ratio`、平均|偏航角| `mean_abs_yaw`。
- 配置 `fusion.subscore.yaw_reliability_deg=45`、`yaw_penalty_cap=0.6`（3D EAR 已缓解
  头转影响，惩罚封顶不清零）。
- 实测：yaw=60° 时眼部可靠度 1.00→0.40，"眼=1 其余=0"场景融合分 0.529→0.310，
  失效模态被自动降权。UI 监测表增"信号质量"行（人脸检出占比着色显示）。

**创新② 眨眼动力学**
- `core/feature_window.py :: _compute_blink_dynamics()`：按时间戳统计窗口内
  平均眨眼时长 `avg_blink_dur` 与微睡眠计数 `microsleep_count`（闭眼段 ≥
  `eye.microsleep_dur_sec=0.5`，Oxford SLEEP Advances 2023 行为微睡眠定义）。
- `core/fusion.py :: eye_subscore()` 新增两路：微睡眠 ≥2 次满分
  （`microsleep_count_full=2`）、平均眨眼时长 0.15→0.5s 线性拉满
  （`blink_dur_normal_sec/blink_dur_full_sec`），与 PERCLOS/闭眼/眨眼率取 max。
- 实测：合成 0.8s 闭眼正确记 1 次微睡眠、平均眨眼时长 0.467s；微睡眠×2 或
  平均眨眼 0.5s 均使眼部子分=1.0。UI 监测表增"平均眨眼(s)""微睡眠(次)"两行。

**创新③ 置信度自适应决策窗口**
- `core/fusion.py :: AlarmFSM.update(level, score, agreement)`：`agreement`（同时
  ≥中度阈的子分数，由 `evaluate()` 统计）≥ `fusion.agreement_for_confident=2` 时，
  所需连续重度窗口数 `3 → max(1, 3−adaptive_alarm_reduction=2) = 1`——证据一致时
  更快报警，证据单薄时维持长窗防误报。
- 实测：一致高分两窗 [True, True]（旧固定窗为 [False, False, True]）；
  弱证据两窗 [False, False]，防误报语义不变。

**创新④ KSS 嗜睡量表对齐**
- `core/fusion.py :: score_to_kss(score)`：`KSS = round(1 + clamp01(S)×8)`，
  融合分线性映射到 Karolinska 嗜睡量表 1..9；`FatigueResult` 新增 `kss` 字段。
- 实测：S=0→KSS1、0.5→KSS5、1.0→KSS9。UI 等级区显示"融合分 x.xxx · KSS n/9"，
  监测表增"KSS 嗜睡度"行（随等级同色）。

**数据结构与配置改动汇总**
- `core/types.py`：WindowFeatures 新增 `avg_blink_dur`/`microsleep_count`/
  `face_ratio`/`mean_abs_yaw`（`current_closed_dur` 为 v1.6 反馈修复已有）；
  FatigueResult 新增 `kss`/`reliabilities`。
- `config.yaml` 新增键均带中文注释：`eye.microsleep_dur_sec`、
  `fusion.adaptive_alarm_reduction`、`fusion.agreement_for_confident`、
  `fusion.subscore.{microsleep_count_full, blink_dur_normal_sec, blink_dur_full_sec,
  yaw_reliability_deg, yaw_penalty_cap}`。
- UI：`ui/panels.py`（等级区 KSS）、`ui/plot_widget.py`（监测表 4 新行）。
  离屏截图确认：等级区/监测表显示正常、文字无裁切。
