# 打包说明 —— 怎么把软件做成 Windows 独立 exe

> 这份是给**做打包的人**看的（你自己，或有 Windows 电脑的组员）。
> 只想使用软件的同学不用看这份，看 `../运行说明.md` 就够了。

## 一句话原理

疲劳检测软件本体只依赖 PyQt5 / OpenCV / MediaPipe / NumPy / SciPy，**不依赖
torch，也不依赖 rPPG-Toolbox 的任何代码**。所以能单独打成一个 exe，不用带上
那个几个 G 的大仓库。

## 谁来打包？打几次？

打包是**开发者做一次**的事，不是测试同学的活。测试同学只负责"下载打好的 exe →
双击 → 录视频测"。你打一次 exe，之后直接把 exe 发给他们即可（改了代码再重打）。

## 一个必须知道的限制

**PyInstaller 不能跨系统打包**——要 Windows 的 exe，就必须在 **Windows** 上打；
在 Linux/WSL 里打出来的是 Linux 程序，不是 .exe。

下面给两种打法，**方式一不需要你自己有 Windows 电脑**，推荐。

---

## 方式一：让 GitHub 自动打包（推荐，无需自己的 Windows）

仓库里已带好 `.github/workflows/build-windows-exe.yml`，GitHub 会用它的 Windows
服务器帮你打。把代码 push 到 GitHub 后：

1. 打开仓库网页 → 顶部 **Actions** 标签 → 左侧选「打包 Windows EXE」→
   右边 **Run workflow** 按钮点一下。
2. 等几分钟（绿勾表示成功）。
3. 点进那次运行，页面最下面 **Artifacts** 里下载 `FatigueDetection-Windows-EXE`，
   解压就是打好的 exe 文件夹。

想让同学**自己去 Releases 页下载**：本地打个版本标签推上去即可自动发布——
```bash
git tag v1.0
git push --tags
```
打完会自动出现在仓库的 **Releases** 页，把链接发同学，他们下载解压双击即可。

---

## 方式二：在本地 Windows 打

适合手边就有 Windows 电脑（你的 WSL 宿主机就是），或不想用 GitHub 的情况。

## 需要拷到 Windows 的东西

只要 `fatigue_system/` 这个文件夹（软件本体 + 本 packaging 目录）。
`experiments/`、`dev_tools/`、`outputs/` 可以不带。整个仓库不用拷。

放成这样的结构就行：
```
疲劳检测系统源码\
└── fatigue_system\
    ├── app.py  config.yaml  core\  io\  ui\
    └── packaging\  (app.spec / build_exe.bat / requirements-app.txt)
```

## 打包步骤

1. 在 Windows 上装 **Python 3.8~3.11**（[python.org](https://www.python.org/downloads/)，
   安装时务必勾选 **Add Python to PATH**）。
2. 进入 `疲劳检测系统源码\fatigue_system\packaging\`，**双击 `build_exe.bat`**。
   （它会自动建独立环境、装依赖、打包，全程约 5~10 分钟，首次要联网。）
3. 跑完后成品在 `dist\疲劳检测系统\`，双击里面的 `疲劳检测系统.exe` 就能用。

不想用 bat 也可以手动敲（在源码根目录）：
```bat
python -m venv build_venv
build_venv\Scripts\activate
pip install -r fatigue_system\packaging\requirements-app.txt
pyinstaller --noconfirm fatigue_system\packaging\app.spec
```

## 分发给测试同学

把 `dist\疲劳检测系统\` **整个文件夹**压成 zip 发过去。他们解压后双击 exe 即可，
**目标电脑不用装 Python、不用装任何依赖**。（exe 旁边的 `_internal` 文件夹是
运行库，不能删、不能单独只发 exe。）

想让同学能自己调参数：把 `config.yaml` 复制一份放到 `疲劳检测系统.exe` **旁边**，
程序会优先读它，改完重启生效，不用重新打包。

## 常见问题

- **杀毒/Defender 报毒**：PyInstaller 打的 exe 偶尔被误报，加信任或允许运行即可
  （这是 PyInstaller 的通病，不是程序有问题）。
- **双击闪退**：多半是 MediaPipe 模型没打进去。确认用的是 `app.spec` 打的
  （里面 `collect_all('mediapipe')` 负责收模型）；实在不行把 `console=False`
  临时改成 `True` 重新打，能看到报错信息。
- **界面中文变方块**：目标机缺中文字体（很少见，Windows 一般自带微软雅黑）。
- **体积偏大（几百 MB）**：正常，MediaPipe 带模型就这么大，已排除 torch 尽量压过了。
