# 疲劳检测与预警系统 —— 参数设置说明

> 本文档覆盖 `config.yaml` **每一个参数**的含义与设置依据（任务书交付物）。
> 使用者只需看第一、二节；第三节是完整参数手册；第四节是调参依据汇总（写报告/答辩用）。

---

## 一、参数文件在哪、怎么生效

- **源码版**：`fatigue_system/config.yaml`，改完**重启程序**生效。
- **exe 版**：把 `config.yaml` 放在 `FatigueDetection.exe` 旁边即可覆盖内置默认值
  （解压包里已附带一份可编辑的），同样改完重启生效。
- **图形界面**：控制区「参数设置」按钮可直接调常用阈值，**立即生效**并可写回
  config.yaml（详见 运行说明.md）。
- 所有参数都带中文注释；本文与 config.yaml 注释一致，以 config.yaml 为准。
- **个性化基线校准会在运行时覆盖部分阈值**（闭眼阈、头部中性角、静息心率），
  这是设计行为（三点延伸之一"个性化基线"）；校准优于手工调参。

## 二、最常调的参数（场景速查）

| 想解决的问题 | 调哪个 | 怎么调 |
|---|---|---|
| 报警太灵敏 / 误报多 | `fusion.alarm_consecutive_windows` | 3 → 4/5（要连续更多秒重度才报警） |
| 报警太迟钝 | 同上；或 `fusion.level_thresholds.severe` | 3 → 2；severe 0.70 → 0.65 |
| 长闭眼直接报警的快慢 | `fusion.microsleep_sec` | 默认 2.0s；设 0 关闭该硬规则 |
| 持续低头报警的快慢 | `fusion.head_down_sec` | 默认 8.0s；常低头看资料的场景可调大或设 0 关闭 |
| 哈欠漏检 | `mouth.mar_yawn_thresh` | 0.30 → 0.22（还有 1.5s 时长兜底，不易误报） |
| 说话被误判哈欠 | 同上 | 0.30 → 0.40 |
| 闭眼检不出（先做校准！） | `calibration.blink_anchor_ratio` / `eye.ear_closed_k_std` | 0.33 → 0.4（锚点阈上移）；k 1.5 → 1.0（无锚点时生效） |
| 睁眼被误判闭眼 | 同上 | 0.33 → 0.25；k 1.5 → 2.0（更保守） |
| 心率数值跳动 | `rppg.hr_smooth_window` / `hr_max_step_bpm` | 9→11 更稳（响应更慢）；12→8 更严 |
| 视频文件想循环播放 | `video.loop_file` | false → true |
| 人脸丢失提示太频繁 | `alarm.face_lost_sec` | 3.0 → 10（设很大≈关闭） |
| 报警没声音/不要弹窗 | `alarm.sound_enable` / `popup_enable` | true/false |

## 三、全参数手册（按 config.yaml 分组）

### 3.1 video —— 视频源（M0）

| 参数 | 默认 | 含义与依据 |
|---|---|---|
| `default_source` | camera | 启动默认源：camera 摄像头 / file 视频文件 |
| `camera_index` | 0 | 默认摄像头设备索引（/dev/video0） |
| `target_fps` | 20 | GUI 帧循环节拍上限。只作节拍，不写入相机（见下）。规格书要求实时 ≥15fps |
| `frame_width/height` | 640/480 | 采集与显示分辨率；越大越清晰但越慢 |
| `loop_file` | false | 视频播完自动停止（组员反馈：便于单遍回放测试）；true 循环 |
| `camera_fourcc` | MJPG | 摄像头编码。**usbip 转发进 WSL2 必须 MJPG**（YUYV 未压缩流带宽过大读帧超时）；原生系统也普遍支持；置空用驱动默认 |
| `camera_capture_thread` | true | 摄像头采集独立线程（持续取流、GUI 只拿最新帧）。间歇式 read() 会把 usbip 摄像头拖到 ~5fps |
| `camera_read_timeout_sec` | 2.0 | 等新帧超时视为相机断开。正常达不到 |

> 注意：**代码故意不对相机设置 CAP_PROP_FPS**——实测对 usbip 摄像头请求 20fps
> 会协商出 ~5fps 的降级模式，保持驱动默认反而最快（交接文档 §4.10）。

### 3.2 ui —— 界面（M0）

| 参数 | 默认 | 含义 |
|---|---|---|
| `font_path` | ""（自动） | 中文字体文件路径。空则自动探测（Windows 微软雅黑 → Noto CJK 等），找不到时中文显示方块 |

### 3.3 facemesh —— 人脸关键点（M1）

| 参数 | 默认 | 含义与依据 |
|---|---|---|
| `max_num_faces` | 1 | 单人检测（任务书场景） |
| `refine_landmarks` | **true** | 虹膜/眼睑精修。**v1.5 关键改进**：CEW 数据集实测开启后睁闭眼 EAR 分离度 0.11→0.27、闭眼检出 78%→96%、总准确率 96.6%。会改变 EAR/MAR 尺度，但闭眼阈由校准自适应、MAR 阈已按新尺度重标 |
| `min_detection_confidence` | 0.5 | 人脸检测置信度 |
| `min_tracking_confidence` | 0.5 | 关键点跟踪置信度 |

### 3.4 eye —— 眼部特征（M1/M2）

| 参数 | 默认 | 含义与依据 |
|---|---|---|
| `ear_closed_thresh` | 0.15 | **未校准时**的绝对闭眼阈（回退值）。v1.9 组员建议 0.21→0.15：CEW 实测把"睁眼被持续误判闭眼、报警不停"的人群从 ~3% 压到 ~1%（误报瘫痪比漏检糟；漏检者校准即愈） |
| `ear_closed_ratio` | 0.6 | 旧方案兼容保留（当前用 眨眼锚点/k×std 统计阈） |
| `ear_closed_k_std` | 1.5 | **校准后无眨眼锚点时**的个性化闭眼阈 = 睁眼EAR中位 − k×(1.4826·MAD)，夹 [50%,92%]×中位。依据 PeerJ 2022 / Soukupová&Čech 2016（固定阈因人而异不可靠）；CEW 实测 k=1.5 检出 99%、误报 6%。捕捉到眨眼锚点时优先用锚点方案（见 3.8） |
| `ear_smooth_frames` | 1 | EAR 滑动平均帧数。1=不平滑：平滑会把 100ms 级短眨眼摊平导致漏检 |
| `blink_min_duration_sec` | 0.06 | 眨眼去抖（按时长，对帧率自适应）。按帧数去抖在低帧率(~8fps)下会让眨眼永远检不出——已踩实的坑 |
| `perclos_window_sec` | 30 | PERCLOS 统计窗口 |
| `blink_window_sec` | 60 | 眨眼率统计窗口 |
| `blink_rate_min_window_sec` | 15 | 眨眼率最小归一化分母：启动时窗口未满除以极小时长会虚高爆表（组员实测 25.7）——宁可早期低估 |
| `microsleep_dur_sec` | 0.5 | 创新②：闭眼段 ≥0.5s 记一次**微睡眠**（Oxford SLEEP Advances 2023 行为微睡眠定义） |
| `eye_valid_pitch_deg` | 30 | v1.9：相对基线低头超过 30° 的帧，眼睑关键点锁到眉毛（组员实测）、EAR 失真——不参与任何眼部统计（既不算闭也不算睁）。0=关闭门控 |

### 3.5 mouth —— 嘴部特征（M1/M2）

| 参数 | 默认 | 含义与依据 |
|---|---|---|
| `mar_yawn_thresh` | 0.30 | MAR 超过且持续即哈欠。本项目 MAR 公式（三垂直对取均值）尺度低于文献"仅中央对"公式，0.6 永远达不到；refine 开启后按哈欠数据集实测（闭口 0.003 / 哈欠中位 0.70）取中间值 0.30 |
| `yawn_min_duration_sec` | 1.5 | 张口持续 ≥1.5s 才算哈欠（过滤说话） |
| `yawn_window_sec` | 60 | 哈欠计数窗口 |

### 3.6 head —— 头部姿态（M1/M2）

| 参数 | 默认 | 含义与依据 |
|---|---|---|
| `pitch_lower_thresh_deg` | 15 | 相对**本人基线**的低头角阈值 |
| `yaw_tilt_thresh_deg` | 20 | 相对基线的偏头角阈值 |
| `nod_window_sec` | 10 | 点头统计窗口 |
| `nod_amplitude_deg` | 8 | 一次点头须"偏离超幅度→再回归"完整摆动（状态机）。原 5° 太敏感，哈欠前倾被连续误判点头（组员反馈#2） |
| `nod_min_count` | 2 | 窗口内点头 ≥2 次才判"点头"状态：单次前倾（哈欠）不算 |

### 3.7 rppg —— 实时心率（M4）

| 参数 | 默认 | 含义与依据 |
|---|---|---|
| `enable` | true | 关闭则生理子分缺失，融合按剩余权重归一化=任务书基础模型 |
| `buffer_sec` | 10 | 滚动 RGB 缓冲长度，POS 的分析窗 |
| `update_interval_sec` | 1.0 | 心率估计节拍 |
| `bandpass_low/high_hz` | 0.7/3.0 | 带通 42~180bpm，滤呼吸/高频噪声 |
| `roi` | cheeks_nose | 肤色 ROI：面颊+鼻部 |
| `resample_fs` | 30 | 相机供帧不均匀，FFT 前按时间戳线性重采样到均匀网格，否则频率轴失真 |
| `pos_win_sec` | 1.6 | POS 滑窗（Wang 2017 论文推荐值） |
| `min_buffer_ratio` | 0.8 | 缓冲不足 80% 不出数（刚启动不给假心率） |
| `max_gap_sec` | 1.0 | 缓冲内出现 >1s 丢脸间断不出数 |
| `hr_smooth_window` | 9 | 逐秒估计的中值滤波窗。抑制谐波伪峰跳变（组员实测 98→57→71）；越大越稳、滞后越长（9≈4-5s） |
| `hr_max_step_bpm` | 12 | 离群门控：偏离近期中位 >12bpm 直接丢弃（心率每秒变不了这么多）。与中值滤波双保险 |
| `auto_rest_min_samples` | 15 | 自动静息心率兜底：不校准时累积 15 个逐秒估计取中位当静息参照，生理子分才能激活（否则永远"-"）。校准值更准、优先 |

### 3.8 calibration —— 个性化基线（M2）

| 参数 | 默认 | 含义 |
|---|---|---|
| `duration_sec` | 30 | 校准采集时长。产出睁眼EAR中位/MAD、眨眼谷底锚点、闭口MAR、头中性角、静息HR |
| `enable` | true | 三点延伸之一，建议保持开启 |
| `blink_anchor_enable` | true | v1.9 组员建议：校准期自然眨眼谷底=本人闭眼水平实测值，比只看睁眼统计外推更准 |
| `blink_anchor_ratio` | 0.33 | 闭眼阈放在"锚点→睁眼均值"区间的此比例处（0.33=**下三分之一**：离睁眼噪声远防误报，又在谷底之上让正常眨眼可过阈被计数） |
| `blink_dip_k_std` | 3.0 | 凹陷判定：低于 睁眼中位−3×稳健std 的连续段视为一次眨眼 |
| `blink_anchor_min_dips` | 2 | 凹陷段数少于 2（校准期没眨眼）→ 无锚点，退回 k×std 公式 |
| `blink_anchor_min_gap_ratio` | 0.25 | 锚点须比睁眼低 25% 以上，否则视为噪声下摆不采用（防误用） |

### 3.9 fusion —— 加权融合与分级（M3，含创新①③）

| 参数 | 默认 | 含义与依据 |
|---|---|---|
| `update_interval_sec` | 1.0 | 融合/报警状态机节拍（按时间戳）。"连续3窗"=3s |
| `weights.eye/mouth/head/physio` | 0.45/0.25/0.15/0.15 | 固定权重 Σ=1（任务书基础模型）。physio 缺失时按剩余归一化（眼 0.529/嘴 0.294/头 0.176）。**创新①在此之上再乘实时可靠度** |
| `level_thresholds.mild/moderate/severe` | 0.25/0.50/0.70 | 四级边界。依据：纯闭眼 S≈0.53 须落"中度"（规格书初始 0.55 会永远到不了）；综合疲劳 S≈0.85 落"重度"；正常噪声 S≲0.15 给 mild 留余量 |
| `smoothing_alpha` | 0.3 | 评分 EMA 平滑系数，越小越平滑 |
| `alarm_consecutive_windows` | 3 | 连续 3 窗重度才报警（任务书防误报要求） |
| `alarm_clear_windows` | 2 | 连续 2 窗低于重度才解除（迟滞防抖） |
| `microsleep_sec` | 2.0 | **硬规则**：当前连续闭眼 ≥2s → 立即重度+报警，越过平滑与连续窗累积（组员反馈#1：微睡眠最危险应立即报）。0=关闭 |
| `head_down_sec` | 8.0 | **硬规则**（v1.9 反馈⑧）：相对基线持续低头 ≥8s → 立即重度+报警。头埋下去打瞌睡时眼部已不可测，持续低头是最后可用证据；阈值比微睡眠长很多（低头还可能是看手机/资料）。0=关闭 |
| `adaptive_alarm_reduction` | 2 | 创新③：证据充分时所需连续窗 3→max(1, 3−2)=1，更快报警 |
| `agreement_for_confident` | 2 | 创新③：同时 ≥中度 的子分数达 2 即"证据充分" |

**fusion.subscore —— 子分归一化锚点（"达到该值记满分"）**

| 参数 | 默认 | 依据 |
|---|---|---|
| `perclos_full` | 0.40 | PERCLOS≥40% 眼部满分（P80 文献重度阈值量级） |
| `eye_closed_full_sec` | 2.0 | 最长闭眼 ≥2s 满分（微睡眠量级） |
| `blink_rate_normal/full` | 20/40 | 眨眼率 20 次/分以内正常，40 满分 |
| `microsleep_count_full` | 2 | 创新②：窗口内微睡眠 ≥2 次满分 |
| `blink_dur_normal_sec/full_sec` | 0.15/0.5 | 创新②：平均眨眼时长 0.15s 正常 → 0.5s 满分（眨眼变长=困倦的文献指标） |
| `yawn_count_full` | 3 | 60s 内哈欠 ≥3 次嘴部满分 |
| `head_lowered_score/tilted_score` | 0.7/0.5 | 持续低头/偏头的固定子分 |
| `nod_count_full` | 3 | 10s 内点头 ≥3 次头部满分 |
| `hr_drop_full_ratio` | 0.15 | 心率较静息下降 ≥15% 生理满分（疲劳时心率下降的文献量级） |
| `yaw_reliability_deg` | 45 | 创新①：\|偏航\|达 45° 时眼部可靠度惩罚到顶 |
| `yaw_penalty_cap` | 0.6 | 创新①：惩罚上限（3D EAR 已缓解头转影响，降权不清零） |
| `pitch_reliability_deg` | 30 | 创新①（v1.9）：相对基线\|俯仰\|达 30° 时眼部惩罚到顶（组员实测低头时 EAR 幅度压缩、检不到闭眼） |
| `pitch_penalty_cap` | 0.8 | 创新①（v1.9）：低头惩罚上限。比 yaw 重（0.8 vs 0.6）：低头是眼睑遮挡+眉毛锁定，比侧转失真更严重 |

## 四、调参依据汇总（报告/答辩可直接引用）

1. **个性化优先于手工调参**：闭眼阈、头中性角、静息心率都由 30s 基线校准按
   "偏离本人正常值"生成（PeerJ 2022：固定 EAR 阈值因人而异不可靠）。
2. **数据集标定**：眼部阈值与 refine_landmarks 用 CEW 整脸睁闭眼数据集实测
   （96.6% 准确率）；MAR 哈欠阈用哈欠视频数据集实测重标（0.30）。
3. **时长判定一律按时间戳**（blink_min_duration_sec 等）：帧数判定隐含 30fps
   假设，在 ~8fps 低帧率下会完全失效（开发中踩实过的坑）。
4. **分级阈值有数学依据**：以"典型场景应落入的等级"反推边界（纯闭眼→中度、
   综合疲劳→重度），而非拍脑袋（见 3.9 表）。
5. **防误报是任务书硬性要求**：EMA 平滑 + 连续 N 窗 + 迟滞清除三重机制；
   创新③在证据充分时自适应缩窗，兼顾灵敏与稳健。
6. **组员实测反馈闭环**：nod_amplitude_deg、blink_rate_min_window_sec、
   hr_smooth_window/hr_max_step_bpm、microsleep_sec、face_lost_sec、loop_file
   六处参数都来自测试组员的实测问题与修复（详见 组员反馈处理记录.md）。
