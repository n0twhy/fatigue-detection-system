# 疲劳检测与预警系统 —— 使用说明

> 这份给**使用/测试这个软件的同学**看，不需要懂编程。照着做就能测。
> 遇到问题看最后的「常见问题」。

---

## 一、这是个什么软件

用一个普通摄像头（或一段录好的视频）看人的脸，实时判断疲劳程度，分成
**清醒 / 轻度疲劳 / 中度疲劳 / 重度疲劳** 四级，重度时弹窗+响铃报警，同时
可以把检测数据存成 Excel 能打开的 CSV 表格。

它怎么判断的（一句话）：同时看三样东西——**眼睛**（闭眼多不多、眨眼频不频、
有没有长时间闭眼）、**嘴巴**（有没有打哈欠）、**头**（有没有低头/偏头/点头），
再加一个**心率**（从脸部皮肤颜色的细微变化算出来，不用戴任何设备），几样加权
打分得出疲劳等级。全程在本机跑，**不上传任何数据、不用穿戴任何设备**。

## 二、怎么打开软件

### 情况 A：拿到的是打好包的文件夹（推荐给测试同学）

1. 解压收到的 `疲劳检测系统.zip`。
2. 进文件夹，双击 **`FatigueDetection.exe`**。
3. 直接就开了，**不用装 Python、不用装任何东西**。

> 旁边那个 `_internal` 文件夹是程序运行要用的，别删、别只拷 exe。

### 情况 B：拿到的是源码（需要先配环境）

需要先装 Python 环境，步骤问开发的同学，或看 `packaging/打包说明.md`。
配好后在源码根目录运行：
```
python -m fatigue_system.app
```

## 三、界面长什么样（六个区）

打开后是一个窗口，分成六块：

```
┌──────────────────────────────────────────────┐
│ ⑤ 顶部工具栏：疲劳检测│视频源▾ │●fps 校准 记录 │
│                        历史 设置 [开始/停止监测]│
├─────────────────────────┬────────────────────┤
│                         │ ③ 疲劳等级          │ ← 大数字+等级+KSS+四项分量条
│   ① 视频显示区           ├────────────────────┤
│   （画面 + 脸上关键点 +   │ ④ 预警提示（一行）   │ ← 状态圆点+文字+累计次数
│     四角信息胶囊）        │                     │
├─────────────────────────┼────────────────────┤
│ ② 指标曲线（近 60 秒）    │ ② 指标列表 / ⑥ 数据 │ ← 点某行→左边切换成该指标曲线
├─────────────────────────┴────────────────────┤
│ 状态行：源 · 分辨率 · fps · 已运行 · 基线状态    │
└──────────────────────────────────────────────┘
```

- **① 视频显示区**：实时画面。脸上画出关键点（眼睛绿、嘴巴红）。四角是信息胶囊：
  右上「人脸 正常/丢失」、左下「EAR·MAR·姿态」、**右下圆钮可开关关键点显示**。
- **② 特征参数区**：右下角**指标列表**（融合分、EAR、PERCLOS、眨眼率、最长闭眼、
  微睡眠、心率、KSS，点「更多指标」展开全部）；**点任意一行，左边曲线就切换成
  该指标的近 60 秒走势**。
- **③ 疲劳等级区**：大数字＝融合分，右侧是等级胶囊（**绿=清醒、黄=轻度、橙=中度、
  红=重度**）和 KSS 嗜睡度；下面四条细进度条＝眼部/嘴部/头部/生理各自的得分
  （**超过中度阈值才变橙**）。
- **④ 预警提示区**：一行状态。正常＝绿点「正常监测中」；报警＝红字「报警中…」并
  累计次数；人脸丢失＝橙字提示回到画面；**疲劳在持续上升时会提前给橙字温和提示
  「疲劳正在累积，建议休息」**（这只是提醒，不响铃、不计入报警次数）。
- **⑤ 操作控制区**：顶部工具栏。左边选视频源，右边四个带字按钮
  **校准 / 记录 / 历史 / 设置**，最右是唯一的实心按钮「开始监测 / 停止监测」。
- **⑥ 数据记录区**：点「记录」后每秒写一行 CSV；停止时**自动生成一份 HTML 会话
  报告**并弹出路径（双击即可打开）。点「历史」可回看历次会话、随时补导出报告。

## 四、基本使用流程

### 第 1 步：选画面来源，点「开始监测」

在顶栏的**视频源下拉框**里选：`摄像头 0`（笔记本自带摄像头一般会自动列出来），
或选「**打开视频文件…**」挑一段录好的 mp4/avi（**测试推荐用视频**，同一段能反复
测、结果稳定）。选好后点最右边的蓝色「**开始监测**」；再点一次（变红「停止监测」）
即停止。

### 第 2 步：校准（重要，务必先做）

对着镜头**保持清醒、睁眼、闭嘴、头摆正、正对镜头**，点顶栏的「**校准**」按钮，
保持约 **30 秒**别乱动（期间正常眨眼即可，不用刻意憋着）。底部状态行显示
「基线已校准」就算好了。

> 为什么要校准：每个人睁眼大小、心率都不同。校准就是先量一遍你正常状态的
> 基准，之后按"偏离你自己的正常值多少"来判断，比用一个固定标准准得多。
> **不校准也能用**（走默认阈值），但校准后更贴合被试本人。

### 第 3 步：看检测结果

盯着③等级区和②特征区。正常状态应该是绿色「清醒」；做出疲劳动作
（见下一节）时，对应数字会变、等级会往上走。

### 第 4 步：记录数据（要导出结果时）

点顶栏「**记录**」按钮（按钮变蓝＝记录中）→ 每秒写一行 CSV。测完再点一次停止，
会生成 **CSV 明细 + 会话汇总 + 一份 HTML 会话报告**（见第六节），并弹窗提示路径。

### 第 5 步：回看历史 / 导出报告

点顶栏「**历史**」→ 左侧列出历次会话，选中即可看到该次的**疲劳等级时间线色带**、
融合分曲线和统计概览；点「**导出报告**」生成单文件 HTML 报告（双击就能打开，
可直接贴进课程报告）。这一切都是读已存的 CSV，**不需要重跑视频、不占摄像头**。

## 五、六个测试场景怎么测（对应课程要求）

每个场景建议单独录一段视频、或摄像头下单独测一次，各点一次「开始记录」导出 CSV：

| 场景 | 怎么做 | 期望看到 |
|---|---|---|
| **正常状态** | 清醒、自然地看镜头 | 绿色「清醒」，**不该报警** |
| **闭眼状态** | 持续闭眼十几秒 | PERCLOS↑、最长闭眼时长↑ → 橙/红，久了报警 |
| **频繁眨眼** | 快速反复眨眼 | 眨眼次数↑ → 黄/橙 |
| **打哈欠** | 张大嘴打哈欠几次 | MAR↑、哈欠数↑ → 黄/橙 |
| **低头** | 低头看向下方 | 头部状态=低头。**短暂低头（看键盘/看资料）不报警**；持续埋头超过 20 秒才判重度 |
| **综合疲劳** | 闭眼+打哈欠+低头一起来 | 红色「重度」→ 弹窗+响铃报警 |

> **关于低头为什么不立刻报警**（v1.10 起）：低头看东西时眼睛会被眼皮挡住一半，
> EAR 数值天然变低——如果照单全收，就会把"低头看键盘"当成"闭眼睡着"狂报警
> （实测 10 分钟能误报 20 次）。所以现在：**低头时眼部数据判为不可信**，且
> **判重度至少要两个互相独立的特征同时异常**（老师建议）。真正的危险状态照旧
> 立刻报：正常坐姿下持续闭眼 2 秒 → 立即重度报警；持续埋头 20 秒 → 立即重度报警。

> 小贴士：测之前**光线要充足、正对镜头**。光太暗会让摄像头变慢、也影响心率和
> 眼睛检测。戴眼镜如果眼睛检测不灵，先做一遍校准。

## 六、结果文件在哪、怎么看

停止记录后，在**输出目录**生成三个文件：

- `fatigue_log_日期时间.csv` —— **逐条明细**（Excel 直接打开不乱码）：每秒一行，含
  EAR、MAR、头部角度、PERCLOS、眨眼率、哈欠数、心率、各子分、疲劳总分、等级、
  是否报警，以及**平均眨眼时长、微睡眠次数、信号质量、KSS 嗜睡度**四项创新指标。
- `session_summary_日期时间.csv` —— **本次汇总**：各等级各占多少时间、报警几次、
  平均帧率。写报告统计准确率/误报率时直接用这个。
- `fatigue_log_日期时间.html` —— **会话报告**（双击打开）：疲劳等级时间线色带、
  各等级时长占比、融合分曲线（带分级阈值线）、PERCLOS/心率曲线、报警时刻、
  基线与关键参数摘要。**可以直接截图贴进课程报告**。忘了导出也没关系——
  点顶栏「历史」，选中任何一次会话都能补生成。

**输出目录在哪**：
- exe 版：在 `FatigueDetection.exe` 旁边的 `outputs\` 文件夹里。
- 源码版：在项目的 `fatigue_system/outputs/` 里。

## 七、界面上的数字都是什么意思（速查）

| 缩写/名称 | 含义 | 大概规律 |
|---|---|---|
| **EAR** | 眼睛张开程度 | 睁眼约 0.3，闭眼接近 0 |
| **MAR** | 嘴巴张开程度 | 闭嘴约 0.02，打哈欠明显变大 |
| **PERCLOS** | 一段时间里闭眼的比例 | 越高越困，是疲劳的核心指标 |
| **眨眼 N次·X次/分** | 眨眼计数和频率 | 过频或过缓都可能是疲劳 |
| **最长闭眼** | 最长一次连续闭眼时长 | 超过 1~2 秒要警惕（微睡眠） |
| **哈欠 N次** | 打哈欠次数 | — |
| **头部** | 正常/低头/偏头/点头 | — |
| **HR** | 心率（bpm） | 静息约 60~90；光线差时可能显示「—」 |
| **融合分 S** | 疲劳总分 0~1 | 越大越疲劳，四级由它划分 |

> 心率显示「—」是正常的：它需要光线好、脸稳定、攒够约 10 秒画面才算得出，
> 算不准时宁可不显示，不给错数。

## 八、常见问题

- **没检测到摄像头 / 打开失败**：换个摄像头选项试试；被别的软件（如相机、
  腾讯会议）占用时要先关掉那些软件。
- **画面很卡、帧率很低**：多半是**光线太暗**，开灯或换亮一点的环境；用录好的
  视频回放则不受影响。
- **一直不报警**：报警是"持续重度好几秒"才触发（防止偶然误报）。要触发就把
  综合疲劳动作**保持久一点**。报警解除也有延迟，属正常。
- **报警没声音**：有的电脑没配音频输出会没声，但红色横幅和弹窗照常有；不影响测试。
- **戴眼镜眼睛检测不准**：先做校准；保证光线足、减少镜片反光。
- **界面文字是方块**：目标电脑缺中文字体（很少见），装一下微软雅黑即可。

## 九、想调参数（进阶，可跳过）

最方便的方式：点控制区的「**参数设置**」按钮，直接调常用阈值（报警灵敏度、
哈欠/闭眼判定松紧、报警声音/弹窗开关等），点「应用」**立即生效**，
点「应用并保存」还会写回配置文件、下次启动继续用。

进一步的全部阈值、权重、窗口长度都在 `config.yaml` 里，每个都有中文注释说明。
exe 版把 `config.yaml` 放到 exe 旁边即可改（改完重启生效）。
每个参数的含义与设置依据见 `参数设置说明.md`。
